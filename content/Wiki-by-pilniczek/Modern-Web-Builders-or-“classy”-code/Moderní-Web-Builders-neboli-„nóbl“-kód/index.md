Když za námi klient přijde s požadavkem na **web**, je velmi důležité pochopit, co přesně potřebuje. Vzhledem k tomu, že weby jsou statické nebo dynamické, přístupy k realizaci projektu mohou být různé, začněme tím rozdílem.

## Dynamické webové stránky

Nejvýraznějším příkladem dynamických **webů** jsou stránky vytvořené na bázi redakčního systému (CMS). Mezi nimi jsou nejčastěji internetové obchody, stejně jako fóra, stránky s recenzemi a další zdroje s možností zveřejňování obsahu návštěvníky.

## Statické webové stránky

Statické webové stránky se používají pro malé a jednoduché webové stránky, které nevyžadují neustálé změny. Stránka má stejný vzhled a stejný obsah pro všechny návštěvníky.

## Přístupy k implementaci

Přístupy k implementaci těchto dvou různých **webů** se liší, stejně jako jejich koncepce.

U dynamického webu bude implementace „nóbl“. V případě statického webu máme kromě psaní kódu ještě jednu možnost: web builder.

## C**o je to web builder**

Web builder je komplexní systém pro tvorbu webových stránek, který můžeme používat i bez znalosti programování. Obvykle běží v prohlížeči.

Pokud chcete web hostovat sami, vybírejte tvůrce webu moudře. Ne každý web builder tuto možnost poskytuje a musíte využít jejich hostingové služby.

V tomto článku si povíme více o statických webech, protože právě pro ně jsou web buildery vhodné.

## Proč **je někdy lepší použít web builder?**

Buďme upřímní... ne vždy je 100% jasné, jaké jsou přesně potřeby klienta.

Řekněme, že designeři mohou použít web builder jako prototypový nástroj, který klientovi pomůže rozhodnout se v krátkém čase.

Statické webové stránky, jako jsou osobní webové stránky a katalogy produktů, lze snadno vytvořit pomocí web builders. Se správným přístupem můžete vystavit projekt v krátkém čase a s malým počtem zdrojů.

## Správný přístup

1. Web builderů je poměrně hodně a je velmi důležité porozumět schopnostem a limitacím každého z nich.
2. **Design. Když se znovu podíváme na omezení každého web builderu, návrh musí být proveden s ohledem na tato omezení.**
3. Pochopte koncového uživatele a podstatu **webu**, nečekejte na webu příliš mnoho interaktivity a dynamiky.
4. **Stojí za to pochopit i velikost projektu. Webová vizitka je pro tvůrce webu jednoduchý úkol, ale něco velkého by bylo lepší implementovat „nóbl“ způsobem.**
5. Osoba s nulovými znalostmi ve vývoji nebude schopna realizovat projekt příliš rychle a správně. Výsledkem každého web builderu je nakonec pouze kód, proto je velmi důležité porozumět funkcím vývoje a optimalizaci webů, stejně jako věcem typu SEO.

Pokud se neřídíte výše uvedenými body, může se web builder ukázat jako dražší a časově náročnější než „nóbl“ přístup.

Pokud jste si ale odpověděli na všechny tyto otázky a chápete, že ve vašem případě je web builder nejlepším řešením, pak určitě pocítíte jeho výhody:

1. je to rychlé,
2. je to pohodlné,
3. úpravy nebo aktualizace jsou snadno proveditelné,
4. je okolo něj velká komunita,
5. má technickou podporu,
6. stačí vám pouze základní znalost o vývoji webu.

## Závěr

Ještě jednou bych ráda zdůraznila, že jen šikovné ruce dokáží postavit skvělý produkt. Jedná se spíše o pomocníka při implementaci malých stránek než o plnohodnotnou náhradu kódování.

Snadný statický web lze bez váhání vytvořit pomocí nástroje pro tvorbu statického webu.

Pokud potřebujete komplexní dynamickou aplikaci, měli byste se této možnosti vyhnout.
