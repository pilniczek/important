## **Všichni**

- dělat rozhodnutí založená na datech
- umí říct "posral jsem to"
- umí říct nevím - nesnaží se mlžit, aby nevypadal za idiota (stejně se pozná, že mlží, a za idiota bude)
- schopný přizpůsobení se změnám / novým technologiím
- Dělat estimace (v závislosti na senioritě), obecně plánovat.
- včas řeší volna/dovolené
- JS
- React
- Typescript
- Next (App router)
- react-query
- react-hook-form
- yup
- HTML
- CSS (MUI i bez dokumentace, s dokumentací Bootstrap/Tailwind) [počítáme, že junior nemusí dát všechno]
- psát/generovat testy (cypress nebo jest)
- Figma (vykopíruje, co potřebuje)
- GIT na úrovni popsané v notionu
- Použít swagger a openapi generátor pro vygenerování kódu, upravení konfigurace
- schopný pracovat jak samostatně tak třeba paircodingem (Umět se podívat do dokumentace dřív, než se bude ptát)
- Komunikovat (informuje o pálení estimace, řeší nejasná zadání, řekne si o paircoding/konzultaci na věci, u kterých si není jistý, veřejně obhajovat svůj názor, koomunikace napříč housy)
- Založit projekt dle specifikace (víme, specifikace je WIP).
- Zprovoznit si vývojové prostředí (node, npm, nvm, git, ide, linting, SSH)
- Správa repa na githubu/gitlabu/bitbucketu/azure = isues, merge, atp. (ono je to všude podobný, není potřeba zvládat do hloubky settings).
- Rozumí package.jsonu (deps, devdeps, scripts, version).
- Nastavit automatizovaný error reporting (Sentry).
- Zareagovat na incident na produkčním prostředí (WIP).
- Namockovat data (../postman/insomnia/tweak).
- Dokumentovat věci (readme, wiki, notion).
- Použít react dev tools a dev tools pro debugging.
- Analyzovat performance webu a navrhnout akční kroky na základě dat z analýzy tak, aby se zlepšila.
- Není pouhý převaděč lidského jazyka do zdrojového kódu - chápe smysl zadání, zná (nebo chce znát) širší kontext, dokáže rozpoznat vzájemně konfliktní zadání, je schopen naprogramovat řešení odpovídající potřebám širšího kontextu. (Nebýt "lopaty"!)
- Udělat codereview - nekomentovat blbosti, umět navrhnout lepší řešení, FAKT TEN KÓD CHÁPAT, NE JEN PROSCROLLOVAT DIFF, podívat se na vybuilděnou verzi.
- vyřešit komunikaci/autorizaci s REST API
